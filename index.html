Give Up Writing Web Pages

You know, we’ve all been through it. Now we understand the box model, quirks mode, creating equal height columns, source-order independence, and what causes the three-pixel jog in IE. Now we spend hours creating complex layouts with floats that work across multiple browsers without using tables. Occasionally, we have to remind ourselves that it isn’t easy for beginners.

However, if you think that learning how to create semantically rich, accessible, standards-compliant, and valid markup just isn’t worth the time–and we know you do–then we have some advice for you: give up writing web pages. Get a professional to do it for you, and spend your time doing something less boring instead.

Problems this site doesn’t have

  • script element as child of html
  • style element as child of html
  • obtrusive use of JavaScript
  • images used for textual content
  • incorrect use of typographical characters
  • invalid sarcasm element
  • all-caps text in source
  • use of invalid attributes
  • punctuation used for visual styling
